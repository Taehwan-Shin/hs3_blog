# 📈 블로그 방문자 수 기능 구현하기: 파이어베이스(Firebase) 가이드

안녕하세요, 신라입니다! 🏔️ 오늘은 블로그의 '생명력'을 시각적으로 보여주는 가장 기초적이면서도 중요한 기능, **방문자 수 집계(Visitor Count)** 기능을 구현한 과정을 공유하려고 합니다.

단순히 숫자를 올리는 것을 넘어, 어떻게 하면 효율적이고 정확하게, 그리고 무료로 이 기능을 유지할 수 있는지 제가 겪은 시행착오를 바탕으로 정리해 보았습니다.

---

## 🧐 왜 방문자 수 기능이 필요한가요?

정성 들여 쓴 글이 얼마나 많은 사람에게 읽히고 있는지 확인하는 것은 창작자에게 가장 큰 동기부여가 됩니다. 또한, 방문자에게는 이 블로그가 얼마나 활발하게 운영되고 있는지 보여주는 신뢰의 지표가 되기도 하죠.

![방문자 수 구현 방법 비교](/posts/attachments/20260514/visitor_count_comparison_3_methods.webp)

---

## 🛠️ 구현을 위한 3가지 방법 후보

처음 기능을 기획할 때 크게 세 가지 방법을 고민했습니다.

1.  **구글 애널리틱스(GA) API 활용**: 가장 정확하지만, 실시간 데이터를 블로그 UI에 직접 노출하기에는 API 설정이 매우 복잡하고 속도가 느릴 수 있습니다.
2.  **자체 백엔드 서버 구축**: DB와 서버를 직접 운영하면 자유도는 높지만, 정적 사이트(SSG)인 제 블로그의 특성상 배보다 배꼽이 더 커지는 상황이었습니다.
3.  **파이어베이스(Firebase) 활용 (선택!)**: 실시간 데이터베이스(Realtime DB)를 지원하며, 설정이 매우 간편하고 일정 사용량까지는 평생 무료라는 점이 가장 큰 매력이었습니다.

---

## 🚀 파이어베이스로 구현하는 단계별 과정

### 1. 파이어베이스 프로젝트 설정
구글 파이어베이스 콘솔에서 프로젝트를 생성하고, **Realtime Database**를 활성화합니다. 보안 규칙은 처음에는 테스트를 위해 읽기/쓰기를 허용하되, 나중에는 특정 경로만 허용하도록 강화해야 합니다.

### 2. Astro 컴포넌트 제작
블로그의 푸터(Footer)나 사이드바에 들어갈 `VisitorCounter.astro` 컴포넌트를 만듭니다.

### 3. 실시간 업데이트 로직 (SDK 활용)
파이어베이스 SDK의 `runTransaction` 함수를 사용합니다. 이는 여러 명이 동시에 접속했을 때 숫자가 꼬이지 않게(Atomic Update) 보장해 주는 핵심 기술입니다.

```javascript
import { ref, runTransaction } from 'firebase/database';

// 방문자 수 1 증가 로직
runTransaction(totalRef, (currentValue) => {
  return (currentValue || 0) + 1;
});
```

## 🚧 [고비] "새로고침할 때마다 숫자가 올라가요!"

기능을 처음 붙였을 때 발생한 가장 큰 문제였습니다. 한 사람이 페이지를 새로고침하거나 다른 글을 읽을 때마다 숫자가 무한정 올라가는 현상이 발생했죠.

### 🔍 원인 분석
클라이언트 사이드에서 아무런 제약 없이 페이지 로드 시마다 `increment` 함수를 호출했기 때문입니다. 이는 정확한 '방문자 수'가 아니라 '단순 페이지 뷰(PV)'에 불과했습니다.

### ✅ 해결 방법: LocalStorage 활용
사용자의 브라우저에 **'오늘 방문 여부'**를 저장하는 로직을 추가했습니다.

1.  사용자가 접속하면 `localStorage`에서 오늘 날짜(`YYYY-MM-DD`)를 확인합니다.
2.  오늘 날짜가 없다면 파이어베이스 숫자를 1 올리고, 현재 날짜를 저장합니다.
3.  이미 날짜가 있다면 숫자를 올리지 않고 현재 DB 값만 가져와서 보여줍니다.

이 간단한 로직 덕분에 **'1인 1일 1회 카운팅'**이라는 정확한 집계가 가능해졌습니다!
