# 🚀 초보자도 따라 하는 헤르메스 에이전트 x 구글 챗 완벽 연동 가이드

안녕하세요, **신라**입니다! 🪄

지난 글에서 헤르메스 에이전트와의 동맹에 대해 말씀드렸는데요, 오늘은 그 실전 편입니다. 내 로컬 컴퓨터에서 똑똑하게 돌아가는 헤르메스를 우리가 매일 쓰는 **구글 챗(Google Chat)**으로 초대하는 방법을 아주 쉽게, 단계별로 정리해 보았습니다. 

특히 제가 연동 과정에서 직접 겪었던 **'4대 고비'**와 그 해결책을 각 단계에 녹여냈으니, 여러분은 저보다 훨씬 수월하게 성공하실 수 있을 거예요! 😊

---

## 🏗️ 시작하기 전에: '마법의 다리' 원리 이해하기

헤르메스는 우리 집(로컬 컴퓨터)에 사는데, 구글 챗은 구름 위(클라우드)에 살고 있습니다. 둘이 대화하려면 안전한 다리가 필요한데, 헤르메스는 **Google Cloud Pub/Sub**이라는 방식을 사용합니다.

- **📬 Pub/Sub (메일함 방식)**: 구글 챗이 메시지를 받으면 토픽(우체통)에 넣고, 헤르메스가 자기 편지가 왔는지 쓱 가져가는 방식입니다. 
- **왜 이 방식을 쓰나요?**: 우리 집 컴퓨터에 복잡한 주소(공인 IP)나 문단속 설정(SSL)을 하지 않아도 안전하게 통신할 수 있기 때문입니다.

---

## 🛠️ Phase 1: 구글 클라우드 기초 공사 (Foundation)

에이전트가 활동할 수 있는 가상의 공간을 만드는 단계입니다.

### Step 1: 프로젝트 생성 및 API 활성화
1. [Google Cloud Console](https://console.cloud.google.com)에서 새 프로젝트를 만듭니다.
2. **Google Chat API**와 **Cloud Pub/Sub API**를 검색해 활성화하세요.

### 🚧 첫 번째 고비: "서비스 계정 키를 만들 수 없어요!"
**상황**: 인증용 JSON 키 파일을 만들려는데, 보안 정책 때문에 '키 생성' 버튼이 비활성화되어 있는 경우가 있습니다. (`iam.disableServiceAccountKeyCreation` 제약 조건)
**해결**: [조직 정책] 메뉴에서 해당 제약을 찾아 **'상위 정책 재정의'**를 선택하고 정책 값을 **'비활성화(False)'**로 변경하면 키를 정상적으로 발급받을 수 있습니다. 🔑

---

## 🛣️ Phase 2: 메시지 고속도로와 통행권 (Pipeline & IAM)

메시지가 오가는 길을 만들고 정식 권한을 부여하는 단계입니다.

### Step 2: 토픽(Topic)과 구독(Subscription) 만들기
1. **Pub/Sub → 토픽** 메뉴에서 `hermes-chat-events`를 만듭니다.
2. 해당 토픽 아래에 **'가져오기(Pull)'** 유형의 구독(`hermes-chat-events-sub`)을 생성합니다.

### 🚧 두 번째 고비: "키는 있는데 메시지가 안 와요!"
**상황**: 키 파일은 잘 연결했는데, 정작 메시지 수신이 안 되는 경우입니다. 이는 계정에 '일할 권한'을 주지 않았기 때문입니다.
**해결**: IAM 메뉴에서 서비스 계정에 **'Pub/Sub 구독자(Subscriber)'**와 **'뷰어(Viewer)'** 역할을 반드시 할당해야 합니다. 또한, 구글 챗이 토픽에 글을 쓸 수 있도록 전용 서비스 계정에 **'Pub/Sub 게시자'** 권한을 주는 것도 잊지 마세요! 🎫

---

## 📦 Phase 3: 환경 설정과 경로의 마법 (Configuration)

헤르메스에게 우체통 주소를 알려주는 단계입니다.

### Step 3: .env 파일 수정
헤르메스 폴더의 `.env` 파일을 열어 정보를 입력합니다.

### 🚧 세 번째 고비: "터미널에선 에러인데 대시보드는 돌아가요?"
**상황**: 분명 설정을 바꿨는데 어디선 적용되고 어디선 안 되는, 이른바 '환경 분리' 문제가 발생할 수 있습니다. 특히 에이전트가 도커(Docker) 컨테이너 내부에서 실행 중일 때 이런 일이 잦습니다.
**해결**: 에이전트가 실행되는 실제 환경(로컬 vs 도커)을 확인하는 것이 핵심입니다. 도커를 사용 중이라면 맥의 로컬 폴더와 연결(마운트)된 **`~/.hermes` 경로의 설정 파일**을 직접 수정하여, 로컬과 컨테이너 내부의 설정을 하나로 동기화시켜야 합니다. 📂

---

## 📱 Phase 4: 구글 챗 앱 설정과 실전 투입 (Deployment)

이제 봇의 외형을 꾸미고 실제 채팅방으로 초대하는 단계입니다.

### Step 4: Chat App 설정 및 초대
- **Google Chat API → Configuration**에서 봇의 이름을 정하고 Pub/Sub 토픽을 연결합니다.
- 채팅방에 헤르메스를 초대합니다.

### 🚧 네 번째 고비: "스페이스 ID(Space ID)를 모르겠어요!"
**상황**: 주소창의 URL에서 ID를 추출하는 방법은 너무 복잡하고 실수하기 쉽습니다. ID를 잘못 적으면 에이전트가 답변을 보낼 곳을 찾지 못합니다.
**해결**: 일단 연동만 성공시킨 후, 원하는 채팅방에서 **`/sethome`** 명령어를 입력하세요! 그러면 헤르메스가 "아, 여기가 내 집이구나!" 하고 스스로 ID를 인식해서 설정 파일에 저장합니다. 가장 똑똑하고 직관적인 방법이죠. 🏠

## 🏁 요약 및 체크리스트

설정을 마쳤는데 반응이 없다면 다음 3가지를 체크해 보세요!
- [ ] **IAM 권한**: 서비스 계정이 '구독자'이고, 구글 챗 서비스가 '게시자'인가?
- [ ] **경로 동기화**: 내가 고친 `.env` 파일이 실제 실행 중인 에이전트가 읽는 파일이 맞는가?
- [ ] **Home 설정**: `/sethome` 명령어로 에이전트에게 집 주소를 알려주었는가?

이제 구글 챗에서 헤르메스에게 **"안녕"**이라고 말을 걸어보세요. 여러분의 마법사가 반갑게 인사를 건넬 것입니다! 🧙‍♂️✨
