# 데이터로 여는 교실의 미래: 워크스페이스 데이터 분석과 자동화

안녕하세요, 고교 삼국시대 세미나의 첫 번째 발표를 맡은 **고구려**입니다. 오늘은 학교 현장에서 생성되는 방대한 데이터를 어떻게 관리하고, 이를 통해 교육적 통찰을 얻을 수 있을지에 대한 실험 과정을 공유하겠습니다.

![슬라이드 1](attachments/20260509/20260509%20고구려%20발표자료_slide_01.png)

이번 발표의 주제는 <span class="highlight-text">구글 워크스페이스 데이터 분석과 자동화</span>입니다. 학교라는 공간은 이제 아날로그를 넘어 엄청난 양의 디지털 데이터를 쏟아내고 있습니다. 하지만 우리는 그 데이터를 제대로 활용하고 있을까요?

![슬라이드 2](attachments/20260509/20260509%20고구려%20발표자료_slide_02.png)

발표는 데이터 수집의 배경부터 구체적인 분석 도구(GCP, BigQuery), 그리고 실제 데이터를 정제하여 교육 현장에 적용하는 과정 순으로 진행하겠습니다.

![슬라이드 3](attachments/20260509/20260509%20고구려%20발표자료_slide_03.png)

저희가 구축한 시스템의 핵심 구조입니다. <span class="highlight-text">구글 워크스페이스</span>에서 발생하는 로그와 문서를 구글 클라우드(GCP)의 **BigQuery**로 모으고, 이를 구글 코랩(Colab)과 제미나이(Gemini)를 통해 분석하는 파이프라인을 구축했습니다.

![슬라이드 4](attachments/20260509/20260509%20고구려%20발표자료_slide_04.png)

현재 데이터 수집은 <span class="highlight-text">약 96% 정도 완료</span>된 상태입니다. 7,300여 건의 파일 데이터를 확보했으며, 이를 통해 학생들의 학습 패턴을 아주 세밀하게 들여다볼 준비를 마쳤습니다.

![슬라이드 5](attachments/20260509/20260509%20고구려%20발표자료_slide_05.png)

구글 워크스페이스 관리자 콘솔에서는 로그 데이터를 최대 180일까지만 제공합니다. 따라서 장기적인 학습 이력 분석을 위해서는 <span class="highlight-text">BigQuery로의 데이터 이관</span>이 필수적입니다. 데이터가 곧 학교의 자산이기 때문입니다.

![슬라이드 6](attachments/20260509/20260509%20고구려%20발표자료_slide_06.png)

BigQuery에 저장된 데이터의 실제 구조입니다. 클래스룸 과제 제출, 드라이브 파일 수정 이력 등 모든 활동이 <span class="highlight-text">고유한 이벤트 로그</span>로 기록됩니다. 하지만 이 데이터는 너무 방대해서 정제 과정이 반드시 필요합니다.

![슬라이드 7](attachments/20260509/20260509%20고구려%20발표자료_slide_07.png)

수만 개의 컬럼 중에서 교육적으로 유의미한 정보(학생 ID, 파일 ID, 활동 시각 등)만을 골라내는 <span class="highlight-text">데이터 마트(Data Mart) 구성</span> 단계입니다. 이 과정을 거쳐야 비로소 AI가 데이터를 읽고 분석할 수 있는 상태가 됩니다.

![슬라이드 8](attachments/20260509/20260509%20고구려%20발표자료_slide_08.png)

추출된 데이터는 CSV 형식으로 가공됩니다. 엑셀보다는 텍스트 기반의 데이터 처리가 AI 에이전트와 연동하기에 훨씬 유리하며, 데이터의 <span class="highlight-text">무결성을 유지</span>하는 데도 도움이 됩니다.

![슬라이드 9](attachments/20260509/20260509%20고구려%20발표자료_slide_09.png)

대량의 데이터를 빠르게 처리하기 위해 <span class="highlight-text">PowerShell의 병렬 처리 기술</span>을 도입했습니다. 수천 명 학생의 데이터를 일일이 순차적으로 처리하는 대신, 여러 개의 프로세스를 동시에 가동하여 분석 시간을 획기적으로 단축했습니다.

![슬라이드 10](attachments/20260509/20260509%20고구려%20발표자료_slide_10.png)

병렬 처리를 통해 통합된 최종 데이터셋입니다. 이제 우리는 특정 학생이 어떤 수업에서 어떤 문서를 언제 작성하고 수정했는지 <span class="highlight-text">한눈에 파악</span>할 수 있는 기반을 마련했습니다.

![슬라이드 11](attachments/20260509/20260509%20고구려%20발표자료_slide_11.png)

이 데이터는 다시 <span class="highlight-text">구글 클래스룸의 수업 리스트</span>와 매칭됩니다. 어떤 선생님의 수업에서 데이터가 활발히 생성되는지, 어떤 과제가 학생들에게 큰 영향을 주었는지 분석할 수 있습니다.

![슬라이드 12](attachments/20260509/20260509%20고구려%20발표자료_slide_12.png)

상세 과제 분석 화면입니다. 학생이 과제를 <span class="highlight-text">몇 번이나 다시 제출했는지</span>, 제출 기한을 얼마나 잘 지켰는지 등의 메타데이터를 추출합니다. 이는 단순한 성취도 평가를 넘어 '학습 태도'를 정량화하는 근거가 됩니다.

![슬라이드 13](attachments/20260509/20260509%20고구려%20발표자료_slide_13.png)

개별 학생의 타임라인 분석 예시입니다. 특정 기간에 어떤 학습 활동이 집중되었는지, 드라이브에서 어떤 문서를 주로 편집했는지 추적하여 <span class="highlight-text">맞춤형 상담 자료</span>로 활용할 수 있습니다.

![슬라이드 14](attachments/20260509/20260509%20고구려%20발표자료_slide_14.png)

이 모든 분석의 최종 목적지 중 하나는 바로 <span class="highlight-text">생활기록부(생기부) 초안 생성</span>입니다. 학생의 활동 이력을 AI에게 전달하면, 누락 없는 객관적 증거를 기반으로 풍성한 문장을 제안해 줍니다.

![슬라이드 15](attachments/20260509/20260509%20고구려%20발표자료_slide_15.png)

앞으로는 이 데이터를 다시 학생들에게 피드백으로 돌려주는 <span class="highlight-text">선순환 구조</span>를 만들고자 합니다. 자신이 어떻게 공부하고 있는지 데이터를 통해 직접 확인하게 하는 것이죠.

![슬라이드 16](attachments/20260509/20260509%20고구려%20발표자료_slide_16.png)

기술적인 구현 방식에 대한 고민도 많았습니다. 클라우드 기반의 분석이 편리하지만, 학교의 민감한 데이터를 처리하기 위해 <span class="highlight-text">로컬 AI 서버(NAS 등)</span>를 구축하는 방안도 검토 중입니다.

![슬라이드 17](attachments/20260509/20260509%20고구려%20발표자료_slide_17.png)

동료 선생님들과 함께한 데이터 분석 연수 현장입니다. AI의 강력함에 모두가 놀라워하셨지만, 동시에 <span class="highlight-text">CLI 기반 도구의 높은 진입장벽</span>도 확인했습니다. 누구나 쉽게 쓸 수 있는 인터페이스가 다음 과제입니다.

![슬라이드 18](attachments/20260509/20260509%20고구려%20발표자료_slide_18.png)

발표를 마무리하며, 학교는 이제 '직관'의 시대를 넘어 <span class="highlight-text">'데이터'의 시대</span>로 가야 한다고 믿습니다. 교사는 더 본질적인 교육에 집중하고, 번거로운 데이터 수집과 초안 작성은 AI 에이전트에게 맡기는 미래를 꿈꿉니다.

## 4. 질의응답 및 토론 (Q&A)

*   **Q1. 방대한 학교 데이터를 처리할 때 비용이나 토큰 문제가 걱정되지 않나요?**
    *   **A**: 구글 워크스페이스 API 호출 자체는 LLM 토큰처럼 과금되는 방식이 아니라 시간당 호출 쿼터(Quota) 제한이 주된 제약입니다. 실제 GCP BigQuery 연동 실험 결과, 비용은 약 73원 정도로 매우 저렴했습니다. 비용보다는 API 속도와 쿼터 관리가 핵심입니다.
*   **Q2. CSV 파일의 한글 깨짐 문제는 어떻게 해결하나요?**
    *   **A**: 엑셀에서 바로 열면 인코딩 문제로 깨질 수 있지만, 메모장이나 텍스트 에디터에서 UTF-8로 확인하면 데이터는 온전합니다. 데이터 분석 시에는 엑셀보다는 파이썬이나 전용 분석 도구를 사용하는 것이 안전합니다.
*   **Q3. 교사 연수에서의 반응은 어떠했나요?**
    *   **A**: 많은 선생님이 AI 에이전트의 가능성에 놀라워하셨지만, '검은색 창(CLI)'에 대해서는 여전히 심리적 거부감과 두려움을 느끼셨습니다. 이를 해결하기 위해 GUI를 갖춘 에이전트(Antigravity 등)나 직관적인 인터페이스를 제공하는 것이 향후 과제입니다.
