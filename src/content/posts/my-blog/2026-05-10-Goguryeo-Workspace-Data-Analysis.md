# 데이터로 읽는 학교: Google Workspace 심층 분석과 자동화의 여정

안녕하세요, 고구려입니다. 🏯

이번 세미나에서는 지난 한 주 동안 제가 집중해온 **"학교 업무와 수업 데이터를 어떻게 수집하고, 분석하며, 자동화로 연결할 것인가"**에 대한 기술적 여정과 교육적 성찰을 공유하고자 합니다. 단순히 로그를 쌓는 것이 아니라, 실제 교사의 업무를 돕고 학생의 성장을 읽어내는 시스템을 구축하는 과정입니다.

---

## 1. 데이터 수집: 180일의 벽을 넘어 BigQuery로

학교의 Google Workspace 데이터는 무궁무진하지만, 기본적으로 180일까지만 저장됩니다. 이를 영구적인 자산으로 만들기 위해 **BigQuery**를 연동하고 데이터 웨어하우스 구축을 시작했습니다.

![데이터 수집 규모](attachments/20260509/20260509%20고구려%20발표자료_slide_03.png)

현재 수집 중인 데이터 규모는 다음과 같습니다:
- **Classroom**: 강좌 약 1,900개, 과제 약 8,700개, 제출물 약 21만 건
- **Chat**: 약 78만 건의 활동 로그 (협업 패턴 분석용)
- **Drive**: 약 95만 개의 파일 인벤토리 및 공유 리스크 추적

이 방대한 로우 데이터를 `workspace_edu`, `workspace_mart`, `workspace_dashboard` 구조로 체계적으로 적재하고 있습니다.

---

## 2. 데이터 마트(Mart)화: 분석 가능한 형태로의 정제

원천 데이터는 헤더 구조가 가변적이고 정제되지 않아 바로 쓰기 어렵습니다. 저는 이를 해결하기 위해 여러 파이썬 스크립트를 제작했습니다.

![기술 산출물](attachments/20260509/20260509%20고구려%20발표자료_slide_04.png)

특히 **`doc_worker_fast.py`**를 통해 Google Docs의 원문과 수정 이력(Revision)을 병렬로 고속 분석하고 있습니다. 단일 워커로는 시간이 너무 오래 걸려 3개의 샤드로 나누어 병렬 처리하도록 개선했습니다. 이를 통해 학생이 문서를 얼마나 자주 수정했는지, 작성 소요 시간은 얼마인지 등을 정량화할 수 있게 되었습니다.

---

## 3. 심층 분석: '제출률'이라는 함정 너머

데이터 분석의 목적은 학생을 감시하는 것이 아니라, **교육적 현상을 투명하게 보는 것**에 있습니다.

![심층 분석 보고서](attachments/20260509/20260509%20고구려%20발표자료_slide_09.png)

### 👻 Ghost Submission (허수 제출) 탐지
과제 제출 상태는 '제출됨(TURNED_IN)'이지만, 실제 첨부 파일이 없거나 내용이 없는 경우를 추출했습니다. 이는 학생의 고의적 누락을 의심하기 위함이 아니라, **과제 설계의 적절성**을 판단하기 위함입니다. 설문형 과제에서는 정상적인 패턴이지만, 보고서형 과제에서 이런 패턴이 나타난다면 교사의 개입이 필요하다는 신호가 됩니다.

### 🔄 Feedback Loop (피드백 루프) 분석
가장 고무적인 성과는 **반려(RETURNED)와 재제출** 사이의 피드백 사이클을 데이터로 확인한 것입니다. 특정 국어 수행평가에서 100건 이상의 반복적인 반려-수정-재제출 패턴이 발견되었는데, 이는 교사의 피드백이 실제 학생의 메타인지적 행동 변화로 이어졌음을 증명하는 강력한 지표입니다.

![피드백 루프](attachments/20260509/20260509%20고구려%20발표자료_slide_11.png)

## 4. 자동화 솔루션: 교사의 '확인 업무' 경감

분석 결과는 보고서로 끝나서는 안 됩니다. 저는 이를 교사의 실무를 돕는 솔루션으로 연결하고자 합니다.

![자동화 워크플로우](attachments/20260509/20260509%20고구려%20발표자료_slide_13.png)

**"주간 과제 확인 알림 봇"**은 다음과 같은 흐름으로 작동합니다:
1. BigQuery에서 이상 징후 후보(Ghost Submission 등) 추출
2. 담당 교사별로 필터링 및 요약
3. Google Chat/Gmail을 통해 교사에게 알림 발송
4. 교사는 알림을 받은 특정 학생의 문서만 선별적으로 확인

이는 AI가 성적을 매기는 위험한 방식이 아니라, 교사가 **'어디를 먼저 열어봐야 할지'**를 알려주는 안전하고 효율적인 자동화입니다.

---

## 5. 마치며: 개인의 생산성을 위한 '녹음 인사이트 앱'

발표를 마무리하며, 개인적인 자동화 실험인 **'녹음 기반 지식 관리 앱'** 구상도 공유했습니다. 모바일 환경에서 떠오른 생각을 녹음하면 Whisper로 전사하고 GPT로 분류하여 제가 사용하는 지식 베이스인 **Obsidian**으로 자동 저장하는 파이프라인입니다.

![녹음 인사이트 앱](attachments/20260509/20260509%20고구려%20발표자료_slide_16.png)

기술은 복잡해 보일 수 있지만, 그 목적은 단순합니다. 파편화된 데이터를 모아 교사의 수고를 덜고, 학생의 성장을 더 선명하게 바라보는 것입니다. 다음 주에는 더 고도화된 분석 결과로 찾아뵙겠습니다.
