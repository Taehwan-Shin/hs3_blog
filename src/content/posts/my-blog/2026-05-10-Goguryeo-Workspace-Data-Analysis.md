# 데이터로 읽는 학교, 분석으로 돕는 교사

안녕하세요, '고교 삼국시대' 세미나의 고구려입니다. 오늘은 학교 현장에서 생성되는 방대한 데이터를 어떻게 관리하고, 이를 통해 교육적 통찰을 얻을 수 있을지에 대한 저의 실험 과정을 공유하겠습니다.

![슬라이드 1](attachments/20260509/20260509%20고구려%20발표자료_slide_01.png)

구글 클라우드와 제미나이(Gemini), 그리고 BigQuery를 활용하여 학교의 다양한 데이터를 끌어오고 분석하는 작업을 시작했습니다. 자동화가 우리에게 어떤 이점을 줄 수 있을지 기대 반, 두려움 반으로 접근했습니다.

![슬라이드 2](attachments/20260509/20260509%20고구려%20발표자료_slide_02.png)

현재 클래스룸에 있는 과제와 문서들을 분석하기 위해 데이터 수집을 시작했고, 오픈 코드를 통해 약 96% 정도 수집이 완료된 상태입니다.

![슬라이드 3](attachments/20260509/20260509%20고구려%20발표자료_slide_03.png)

한 가지 중요한 사실은 구글 워크스페이스 클라우드에 로그가 180일까지만 저장된다는 점입니다. 누적된 데이터를 장기적으로 보려면 반드시 BigQuery 연동 기능을 미리 켜두어야 합니다.

![슬라이드 4](attachments/20260509/20260509%20고구려%20발표자료_slide_04.png)

실제로 끌어온 클래스룸 데이터의 모습입니다. 헤더가 엄청나게 많고 복잡해서 이대로는 쓸 수가 없습니다. 필요한 것만 정제하는 과정이 반드시 필요합니다.

![슬라이드 5](attachments/20260509/20260509%20고구려%20발표자료_slide_05.png)

정제를 위해 원문만 가져오는 것이 아니라, 학생들의 문서 수정 횟수나 복사 붙여넣기 횟수 같은 정량적인 데이터까지 추출하도록 AI에게 요청했습니다. 

![슬라이드 6](attachments/20260509/20260509%20고구려%20발표자료_slide_06.png)

처리할 데이터가 너무 많아서 AI 에이전트가 스스로 판단하여 PowerShell로 병렬 처리 코드를 짜서 돌리고 있습니다. 이런 식으로 자동화를 시켜놓으니 작업 시간이 크게 단축되었습니다.

![슬라이드 7](attachments/20260509/20260509%20고구려%20발표자료_slide_07.png)

AI가 추천해 준 개념 중 하나가 '데이터 마트(Mart)화'입니다. 원천 데이터를 AI나 명령어가 꺼내기 쉽도록 중간 테이블 형태로 정리해 두는 작업이 필수적입니다.

![슬라이드 8](attachments/20260509/20260509%20고구려%20발표자료_slide_08.png)

데이터는 CSV 파일 형태로 로컬 PC에 저장됩니다. 엑셀로 열면 인코딩 문제로 한글이 깨져 보이지만, 메모장이나 텍스트 에디터로 열면 데이터가 온전하게 유지되는 것을 확인할 수 있습니다.

![슬라이드 9](attachments/20260509/20260509%20고구려%20발표자료_slide_09.png)

데이터 안에는 클래스룸 ID, 과제 ID, 문서 ID 등이 모두 들어있습니다. 나중에 App Script를 호출할 때 굳이 구글 시트에서 ID를 찾지 않아도 이 데이터에서 바로 끌어다 쓸 수 있습니다.

![슬라이드 10](attachments/20260509/20260509%20고구려%20발표자료_slide_10.png)

가장 우려했던 API 호출 비용 문제인데, 실제 GCP 청구서를 확인해보니 73원밖에 나오지 않았습니다. 시간당 호출 쿼터 제한이 있을 뿐, 문서 추출 자체는 거의 무료에 가깝게 진행할 수 있었습니다.

![슬라이드 11](attachments/20260509/20260509%20고구려%20발표자료_slide_11.png)

에이전트와 대화하며 수업 리스트를 뽑아달라고 하면, 제가 참여한 수업 목록과 과제들을 금방 JSON 파일 형태로 매칭해서 정리해 줍니다. 

![슬라이드 12](attachments/20260509/20260509%20고구려%20발표자료_slide_12.png)

문서 내의 텍스트와 제출 상태뿐만 아니라, 구글 드라이브 API를 통해 작성 소요 시간이나 접속 시간까지 추적할 수 있습니다. 단기간에 내용이 급증한 문서도 식별이 가능합니다.

![슬라이드 13](attachments/20260509/20260509%20고구려%20발표자료_slide_13.png)

이러한 분석을 통해 학생의 수행평가 참여도를 아주 세밀하게 볼 수 있습니다. 교사가 피드백을 주고 반려(Return)했을 때, 학생이 어떻게 수정해서 재제출했는지 그 '피드백 루프'를 추적할 수 있습니다.

![슬라이드 14](attachments/20260509/20260509%20고구려%20발표자료_slide_14.png)

보안이나 개인정보 문제가 우려된다면, 클라우드가 아닌 로컬 AI 서버 환경을 구축하는 것도 방법입니다. 특정 시간마다 로컬로 다운받게 설정해 두면 비용과 보안 문제를 동시에 해결할 수 있습니다.

![슬라이드 15](attachments/20260509/20260509%20고구려%20발표자료_slide_15.png)

데이터가 정리되면 생기부(생활기록부) 작성을 위한 초안 생성도 자동화할 수 있습니다. 프롬프트를 튜닝하여 모든 학생에 대한 맞춤형 평가를 대량으로 생성하는 워크플로우를 준비 중입니다.

![슬라이드 16](attachments/20260509/20260509%20고구려%20발표자료_slide_16.png)

이런 데이터를 한눈에 볼 수 있는 대시보드 형태의 확인 업무 경감 솔루션도 구상하고 있습니다. 직관적인 UI를 제공하면 교사들의 접근성이 훨씬 높아질 것입니다.

![슬라이드 17](attachments/20260509/20260509%20고구려%20발표자료_slide_17.png)

며칠 전 교내 선생님들을 대상으로 관련 연수를 진행했습니다. 오픈 코드를 활용하는 방법을 보여드렸는데, 많은 분들이 큰 관심을 보이셨습니다.

![슬라이드 18](attachments/20260509/20260509%20고구려%20발표자료_slide_18.png)

다만, 선생님들이 검은색 터미널 창(CLI)에 대해 큰 두려움을 느끼는 것을 확인했습니다. 명령어를 직접 쳐야 한다는 부담감을 줄여주는, 안티그래비티와 같은 직관적인 GUI 에이전트의 필요성을 다시 한번 실감했습니다. 이상입니다.
